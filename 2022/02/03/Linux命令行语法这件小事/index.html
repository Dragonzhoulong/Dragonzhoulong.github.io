Man 手册是干甚的 里面有些甚

explore Man Linux ，之类的字眼（突然发现之前嗯看某本Unix高级编程PDF，看了好几天的东西其实自己就看了个大概）

man手册[^man]

“man” 是英文单词manual的缩写，中文意为手册、帮助文档，在CentOS 6.8系统中，调用 man man命令，会显示“格式化在线帮助文档”的字样。

man手册章节

标准的man手册主要分为8个章节，分别为：

1 User Commands // 用户命令

2 System Calls // 系统调用

3 C Library Functions // C函数库调用

4 Devices and Special Files // 设备文件和特殊文件

5 File Formats and Conventions // 配置文件及格式

6 Games et. Al. // 游戏 ?

7 Miscellanea // 杂项

8 System Administration tools and Deamons // 管理类命令

注意：这里所说的man手册是指标准的man手册，不同版本的Linux发行版可能会在自己的man手册中新增一些内容。

帮助手册中的段落说明：
NAME 名称及简要说明
SYNOPSIS 用法格式说明
• [] 可选内容
• <> 必选内容
• a|b 二选一
• { } 分组
• … 同一内容可出现多次
DESCRIPTION 详细说明
OPTIONS 选项说明
EXAMPLES 示例
FILES 相关文件
AUTHOR 作者
COPYRIGHT 版本信息
REPORTING BUGS bug信息
SEE ALSO 其它帮助参考

所以如果说，你想要查一下某个C库函数比如说getopt这个获取命令行参数的函数，但是你man getopt 查询出的确实一个命令的参数

因此你应该

看懂命令行参数提示的意思

参考的博客和网站[^a]

docopt—language for description of command-line interfaces

docopt用于为命令行应用定义接口，并且自动生成解析器

参数（位置参数和选项的参数）

  1. 位置参数<argument>或者ARGUMENT

以< >或者大写字母被解释为位置参数

  1. 选项的参数-o argumentor--option=argument中的argument紧跟在选项后面的参数，用于指定选项的操作对象

(int argc,char* argv[])

argc是字符组的数字

"input:./program.exe -a -b"=>argc=3

argv是字符串的指针

argv[i]=>"input: ./program.exe -a -b" "output: -a -b"

空格分开俩个字符组，argv[0]=>./program.exe即程序名

input:"./program.exe -a -b"=> - a - b
